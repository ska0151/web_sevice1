# GitHubプッシュ禁止ファイルと除外設定マニュアル

このマニュアルでは、GitとGitHubを使用する際に、誤って公開リポジトリにプッシュしてはいけないファイルの種類と、それらを自動的にプッシュから除外するための方法（`.gitignore`）について解説します。

## 1. GitHubにプッシュしてはいけないファイル

以下の種類のファイルは、セキュリティ、プライバシー、またはプロジェクトの管理上の理由から、GitHubのような公開リポジトリにはプッシュすべきではありません。

1.  **機密情報を含むファイル**
    *   **`.env`ファイル**: APIキー、データベースの認証情報、パスワードなどの環境変数を格納するファイルです。これは最も重要です。
    *   設定ファイル（例: `config.ini`, `settings.py`など）の中に直接書かれたAPIキーやパスワード。
    *   証明書ファイル (`.pem`, `.crt`, `.key`など)。
    *   トークン、シークレットキーなど。

2.  **仮想環境のファイル**
    *   `venv/`, `env/`などの仮想環境フォルダ全体。これらのファイルはマシン固有のものであり、`requirements.txt`があれば再構築可能です。コードの肥大化も防ぎます。

3.  **コンパイルされたファイルや自動生成ファイル**
    *   Pythonのバイトコードファイル (`__pycache__/`, `*.pyc`)。これらはPythonが自動生成します。
    *   ログファイル (`*.log`)。
    *   データベースファイル (`*.sqlite3`, `db.sqlite3`など)。
    *   テスト結果やカバレッジレポートの出力 (`.coverage/`, `htmlcov/`)。

4.  **IDE（開発環境）固有のファイル**
    *   `settings.json`などのVS Code設定ファイル (`.vscode/`フォルダ内)。チームで共有すべき設定は除く。
    *   `.idea/`（PyCharmなどJetBrains系IDEのフォルダ）。

5.  **一時ファイルやOS固有のファイル**
    *   一時ファイル (`*.tmp`, `*.~`)。
    *   macOSの`.DS_Store`、Windowsの`Thumbs.db`など。

## 2. プッシュしたくないファイルを明示的に禁止するアプローチ（`.gitignore`）

Gitでは、リポジトリのルートディレクトリに**`.gitignore`**という名前の特殊なファイルを作成することで、指定したファイルやフォルダをGitの管理対象から除外（無視）することができます。これにより、`git add .`や`git commit`を実行しても、これらのファイルは誤ってプッシュされることがなくなります。

### `.gitignore`ファイルの作成と記述方法

1.  **`.gitignore`ファイルの作成**: 
    プロジェクトのルートディレクトリに`.gitignore`という名前でファイルを作成します。（例: `c:\Users\ryota\OneDrive\OneDrive_Store\03_Programming\09_web\.gitignore`）
    もし既に存在する場合は、そのファイルを編集します。

2.  **記述ルール**: 
    各行に無視したいファイルやディレクトリのパターンを記述します。
    *   **ファイル名**: `my_secret.txt`
    *   **ディレクトリ名**: `venv/` (ディレクトリ名の場合、末尾に`/`を付けるのが一般的)
    *   **特定の拡張子のファイル**: `*.log` (すべての`.log`ファイルを無視)
    *   **特定のディレクトリ内の特定のファイル**: `logs/*.log` (`logs`ディレクトリ内のすべての`.log`ファイルを無視)
    *   **特定のディレクトリ内のすべてのファイル**: `tmp/*` (`tmp`ディレクトリ内のすべてのファイルを無視)
    *   **特定のディレクトリを再帰的に無視**: `**/__pycache__/` (どの階層にあっても`__pycache__`ディレクトリを無視)
    *   **コメント**: `#`で始まる行はコメントとして扱われます。

### 推奨される`.gitignore`の例

あなたのPythonプロジェクトの場合、以下のような内容を`.gitignore`ファイルに記述することを強く推奨します。

```gitignore
# 環境変数ファイル
.env

# 仮想環境
venv/
env/

# Pythonコンパイル済みファイル
__pycache__/
*.pyc

# IDE関連ファイル
.vscode/
.idea/

# ログファイル
*.log

# データベースファイル
*.sqlite3
db.sqlite3

# 一時ファイル
*.tmp
*~

# OS固有のファイル
.DS_Store
Thumbs.db
```

### `.gitignore`の適用後

`.gitignore`ファイルを編集・保存したら、`git status`コマンドを実行して、無視したいファイルがGitの管理対象外になっていることを確認してください。

もし、以前に誤ってプッシュしてしまったファイルを`.gitignore`に追加した場合、そのファイルは履歴から消えません。履歴からも削除したい場合は、別途Gitのコマンド（`git rm --cached ファイル名`など）を使用する必要がありますが、これは少し複雑な操作なので、まずは`.gitignore`で今後プッシュしないように設定することが重要です。

**常に`.env`ファイルと仮想環境フォルダは`.gitignore`に追加し、機密情報をコードに直接書き込まないように徹底しましょう。**
