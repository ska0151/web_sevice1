# Python仮想環境マニュアル

このマニュアルは、Python仮想環境の基本的な概念、作成方法、利用方法、そして注意点について、初心者の方にも分かりやすく解説します。

## 1. 仮想環境とは何か？なぜ必要なのか？

Pythonで複数のプロジェクトを開発していると、それぞれのプロジェクトで異なるバージョンのライブラリ（`fastapi`や`requests`など）が必要になることがあります。

例えば、あるプロジェクトAでは`requests`のバージョン1.0が必要なのに、別のプロジェクトBでは`requests`のバージョン2.0が必要、といった状況です。Pythonを普通に使うと、これらのライブラリはすべて「グローバル（全体）な環境」にインストールされてしまいます。こうなると、バージョンが衝突して、どちらかのプロジェクトが動かなくなってしまう、という問題が発生します。

**仮想環境（Virtual Environment）**は、この問題を解決するためのものです。

-   **独立した作業スペース**: プロジェクトごとに「隔離された小さなPython環境」を作成します。
-   **依存関係の分離**: それぞれの仮想環境に、そのプロジェクトで必要なライブラリだけをインストールできます。これにより、他のプロジェクトのライブラリと衝突することがなくなります。
-   **クリーンな管理**: プロジェクトが不要になったら、仮想環境ごと削除するだけで、関連するライブラリもきれいにアンインストールできます。グローバル環境がごちゃごちゃになりません。

「プロジェクトごとに専用の作業部屋を用意する」と考えるとイメージしやすいでしょう。

## 2. 仮想環境の作成方法

Pythonには`venv`という仮想環境を作成するための標準モジュールが組み込まれています。特別なツールのインストールは不要です。

1.  **プロジェクトフォルダへの移動**: まず、仮想環境を作成したいプロジェクトのフォルダ（例: `c:\Users\ryota\OneDrive\OneDrive_Store\03_Programming\09_web`）にターミナルで移動します。

    ```bash
    cd c:\Users\ryota\OneDrive\OneDrive_Store\03_Programming\09_web
    ```

2.  **仮想環境の作成**: 以下のコマンドを実行します。

    ```bash
    python -m venv venv
    ```
    -   `python`: Pythonインタープリタを指します。
    -   `-m venv`: `venv`モジュールを実行することを意味します。
    -   `venv`: 作成する仮想環境の名前です。慣習的に`venv`とすることが多いですが、`myenv`など好きな名前に変更できます。このコマンドを実行すると、指定した名前のフォルダ（この場合は`venv`フォルダ）がプロジェクト内に作成されます。

## 3. 仮想環境の有効化 (アクティベート)

仮想環境を作成しただけでは、まだグローバルなPython環境が使われています。作成した仮想環境を「使い始める」ためには、有効化（アクティベート）する必要があります。

### Windowsの場合

```bash
.\venv\Scripts\activate
```

### macOS / Linuxの場合

```bash
source venv/bin/activate
```

**有効化されたことを確認する方法**: 
ターミナルのプロンプトの先頭に、仮想環境の名前（例: `(venv)`）が表示されます。これが見えていれば、仮想環境が有効になっています。

```
(venv) PS C:\Users\ryota\OneDrive\OneDrive_Store\03_Programming\09_web>
```

## 4. 仮想環境の利用方法

仮想環境が有効化されたら、`pip install`コマンドでインストールするライブラリはすべてその仮想環境内にのみインストールされます。

-   **ライブラリのインストール**: `requirements.txt`ファイルに書かれたライブラリを一括でインストールする場合。

    ```bash
    (venv) pip install -r requirements.txt
    ```

-   **個別のライブラリのインストール**: 

    ```bash
    (venv) pip install ライブラリ名
    ```

-   **Pythonスクリプトの実行**: 

    ```bash
    (venv) python main.py
    ```
    このように実行することで、仮想環境内にインストールされたPythonとライブラリが使用されます。

## 5. 仮想環境の無効化 (ディアクティベート)

仮想環境での作業を終え、グローバル環境に戻りたい場合は、以下のコマンドを実行します。

```bash
deactivate
```

ターミナルのプロンプトから `(venv)` の表示が消えれば、無効化されています。

## 6. 仮想環境の削除方法

プロジェクトが不要になった場合や、仮想環境を作り直したい場合は、単に仮想環境のフォルダを削除するだけでOKです。

1.  **仮想環境を無効化する**: `deactivate` コマンドで仮想環境を無効化します。
2.  **仮想環境フォルダを削除する**: プロジェクトのルートディレクトリにいる状態で、以下のコマンドを実行します。

    ```bash
    rmdir /s /q venv
    ```
    (Windowsの場合)

    ```bash
    rm -rf venv
    ```
    (macOS / Linuxの場合)

    これで、`venv`フォルダとその中身（仮想環境にインストールしたすべてのライブラリを含む）が完全に削除されます。

## 7. 注意事項

-   **プロジェクトごとに作成**: 基本的に、新しいPythonプロジェクトを開始する際には、そのプロジェクト専用の仮想環境を作成することをお勧めします。
-   **.gitignoreへの追加**: 仮想環境のフォルダ（例: `venv/`）は、Gitなどのバージョン管理システムには含めないのが一般的です。`.gitignore`ファイルに`venv/`と追記しておきましょう。
-   **Pythonのバージョン**: 仮想環境は、作成時に使ったPythonのバージョンを引き継ぎます。例えば、Python 3.9で`venv`を作成すると、その仮想環境内ではPython 3.9が使われます。
-   **IDEとの連携**: VS Codeなどの多くのIDEは、プロジェクト内に`venv`フォルダがあると自動的に仮想環境を検出し、利用するように設定してくれます。手動で設定する必要がある場合は、IDEのドキュメントを参照してください。

仮想環境を適切に利用することで、Python開発がより快適で管理しやすくなります。ぜひ活用してください。
